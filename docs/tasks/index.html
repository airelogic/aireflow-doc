    May 26, 2021

    What is a Task?

    A task is an actionable output from a Workflow that requires completion by a user in order to fulfill the workflow requirements.

    A task takes a Name, that is required, and optionally a Description and Assignee. If included, the Assignee should reference a specific user.

    Activity Types

    Tasks have 3 types that they can be at present: General, CompleteForm, and ReviewForm.

    General

    This is an all-purpose type that allows for any task.

    CompleteForm

    This task is used to indicate that a specific form should be completed and submitted. possible issue: form selection does not appear

    ReviewForm

    This task is used to indicate that a submitted form should be reviewed. This may be for a secondary opinion, or a managerâ€™s overview, for example.

    Notifications

    A notification is a way of alerting a user or system that a task has been created and should be actioned. Notifications can be sent via Email or HTTP. Multiple notifications can be configured for any given task.

    Email

    An email notification is as it sounds. An email is sent to a user (perhaps defined in the input data). While intended to inform the user that there is a task they need to complete, it can also be used to send receipt of a submission, a summary reply, or any other email. For example, a manager might receive an email notifying them of a ticket that has been raised, while the customer might receive a notification that their issue has been assigned to a manager to look into.

    HTTP

    An HTTP notification is a simple POST request to a defined url. The body of this request can be formed ti cinlude data and match any required shape for the receipient endpoint.

    State

    State can be added to a task to provide users the ability to add more details when completing tasks. It takes the form of a simple Kay-Value set, with the option of setting a default value when the task is created. This is primarily useful for containing systems that integrate with AireFlow.

    Filters

    Filters can be added to tasks to determine if the task should be created. These should be an XPath expression that results in either true (if the task should be created) or false (if it shouldnâ€™t).
