    AireFlow Terminology

    Learn some of the basic terminology of AireFlow

    Mar 23, 2021

    Workflow Definitions

    Workflow definitions can be thought of as templates that describe the structure of a workflow you may want to run at some point in the future. Definitions can contain XPath and Liquid expressions that will be evaluated against any input given upon instantiation. More details about mapping are available here

    Workflow Instances

    Workflow instances are created from definitions and can be provided with input upon creation. Each instance has a Subject ID, that can either be pulled from the input using an expression or specified explicitly when instantiating a workflow.

    Tasks

    Tasks are defined as part of a workflow definition, then created within the workflow instance. Tasks can have a state made up of key/value pairs, which can be updated via the runtime API or runtime UI. Tasks can also send notifications when a workflow instance is run.

    Running Workflows

    When a workflow instance is created, it isnâ€™t always run immediately. When an instance is triggered, it runs by sending any associated notifications for all tasks. When running a workflow, all tasks are run simultaneously, meaning all notifications for all tasks are sent as soon as a workflow is run (tasks are not dependent on each other being completed or even started). If you need one task to run after another, that task should be part of a separate workflow with an event trigger.

    Triggers

    Triggers allow you to specify when a workflow instance should be run. Triggers can optionally instantiate a workflow if it does not already exist, alternatively they can run an existing instance. There are 4 types of triggers:

    • Once - Run the workflow once, either immediately or at 1 fixed time in the future.
    • Event - Run the workflow when some other event happens (e.g. another workflow is completed, or a task is started)
    • Scheduled - Repeatedly run the workflow at a definied frequency
    • HTTP - Run the workflow when a certain http request is received

    Streams

    Streams can be used to create workflow instances based on external events, such as a forms4health form being submitted.
